## 1. User request

Please help me build a Python3 GTK3 application, called **QDVC Equip**.

**Context:** This is a desktop application to help the user keep track of their tools, equipment, and materials for home, work, travels, etc.

**Data:** The data for a particular context (e.g., home, office) is stored inside a folder (in this application, such a folder is called a "workspace"). Nested folders inside a workspace identify where a particular tool, equipment, object, or set of materials is stored. Each tool, piece of equipment, or set of materials is called an "asset". Each asset's information is stored as a YAML file.

Example:

    - ~/qdvc-equip/home (workspace)
        - kitchen (folder)
            - pantry (folder)
                - coffee_machine.yml
                - blender.yml
                - sony_headphones_2.yml
        - study (folder)
            - study_desk (folder)
                - sony_headphones_1.yml
                - keychron_k1_keyboard.yml
    - ~/qdvc-equip/office (workspace)
        - work_desk (folder)
            - sony_headphones_3.yml
Observations:

1. There is no inherent mapping between the depth of a folder and real-world ontology. For example, work_desk is at the top level of the "~/qdvc-equip/office" workspace, but study_desk is nested inside the top-level folder study in "~/qdvc-equip/home".

2. There is an effort to keep the underlying filesystem following a naming convention (essentially snake_case). No spacebars, no special characters, etc.

3. No two folders or assets within a workspace should have the same name, so you see sony_headphones_1.yml and sony_headphones_2.yml.


**Visual layout:** The application is intended to have the look and feel of a typical GNOME2-era application: menubar, toolbar, three panes, and a status bar.

- Menubar: File, Edit, View, Help, triggered by Alt+F, Alt+E, Alt+V, Alt+H.
- Toolbar: New tab, New asset, Save asset, Card view, Read-only, Preview
- Three panes:
    - Navigation tree (pane 1)
    - Items (pane 2)
    - Item details (pane 3)
- Status bar: Indicates current status information

There are many similarities between this visual layout and the codebase at https://github.com/blairwang-online/qdvc-markdown-notebook but with the following notable differences:

- QDVC Equip should be able to open multiple workspaces at the same time. So, whereas qdvc-markdown-notebook has a single "Subfolders" entry in the navigation tree, QDVC Equip would have such an entry for each open workspace.
- QDVC Equip does not, at this stage, have a fourth pane (qdvc-markdown-notebook uses that for a document outline)
- QDVC Equip renders nested subfolder hierarchies within the workspace (important for visually indicating where an asset is stored); the qdvc-markdown-notebook only renders one level of subfolders.
- QDVC Equip works with YAML files, qdvc-markdown-notebook works with Markdown files. The behaviour is similar when viewing the underlying plaintext (the default view), but the "Preview" button in the toolbar is very different. For qdvc-markdown-notebook, "Preview" means to render the Markdown file into rich text, which is still fundamentally text. For QDVC Equip, "Preview" means to replace the text view with UI labels, something like:

```
☕ Coffee Machine <-- this part in large text

```

The button [copy] should copy the relevant data to the clipboard.
